# Transparency

## Funding

FACTION has no external funding. It is an independent project with no investors, donors, grants, or advertising revenue. There are no financial relationships with any organisation whose activities FACTION covers.

## Technology

FACTION is built with Next.js (React), deployed on Vercel, and stores all fact files as JSON documents in a GitHub repository. The full codebase is available for inspection. There is no proprietary algorithm determining what gets covered or how it is ranked.

Source priority ranking on the home page is determined by community votes — a simple counter stored per fact file. There is no editorial weighting of the ranking.

## Known limitations

**Geographic scope.** FACTION currently covers primarily US-based events where government document disclosure laws (FOIA, court e-filing systems, agency publication requirements) create accessible primary source records. Coverage of events in jurisdictions with less document disclosure is limited by source availability, not editorial choice.

**Document availability lag.** Official investigations, court proceedings, and agency reports take time to produce. FACTION cannot document facts faster than the official record materialises. Files in "Open" or "Developing" status reflect the current state of the documentary record, not a complete picture of events.

**Link rot.** Government URLs change. Court filing systems are reorganised. FACTION runs automated daily link health checks and manually verifies sources on a rolling basis, but some links may temporarily break between checks. Use the [link health admin page](/admin/link-health) to check current status, or [submit a correction](/submit?type=correction) if you find a broken link.

**Language.** All FACTION content is currently in English. Primary sources in other languages are not currently indexed.

**Selection bias.** Every topic selection is a judgment call made against the criteria in the [methodology](/methodology). Reasonable people can disagree about what meets the threshold for significance. FACTION publishes its selection criteria transparently so this judgment can be challenged.

## Corrections policy

All factual corrections are:
1. Made promptly upon discovery
2. Logged publicly in the affected fact file's Corrections & Updates section
3. Described specifically — "what was wrong, what it has been changed to"
4. Never quietly deleted without a record

FACTION does not silently edit fact files. If something changes, there is a changelog entry.

## Conflicts of interest

The editorial team discloses any financial or personal relationship with an organisation appearing in a fact file. If such a relationship exists, a note is added to the relevant fact file. No such relationships currently exist for any published file.

## Contact

Corrections, source suggestions, and editorial questions can be submitted via the [submission form](/submit). The editorial team reviews all submissions and responds to those requiring clarification.
